# Kronos 云服务器部署教程

## 环境要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Ubuntu 20.04 / 22.04 / 24.04 |
| Python | 3.9 及以上（脚本自动安装） |
| 内存 | 建议 2GB 以上（加载模型需 4GB+）|
| 端口 | 7070（Flask）、80（Nginx，可选）|

---

## 一、连接服务器

在本地打开终端（Windows 用 PowerShell 或 CMD）：

```bash
ssh root@你的服务器IP
```

---

## 二、上传项目到服务器

**方式 A：压缩包上传（推荐）**

本地 PowerShell 执行：
```powershell
# 压缩项目
Compress-Archive -Path Kronos_Deploy -DestinationPath kronos.zip

# 上传到服务器
scp kronos.zip root@你的服务器IP:~
```

服务器上执行：
```bash
apt install unzip -y
unzip kronos.zip
```

**方式 B：Git 克隆**

```bash
git clone 你的仓库地址 Kronos_Deploy
```

---

## 三、一键部署

```bash
cd ~/Kronos_Deploy
sed -i 's/\r//' deploy/setup_server.sh
bash deploy/setup_server.sh
```

> 脚本在 Windows 上编辑过，`sed` 命令用于去除换行符兼容问题，必须执行。

脚本自动完成：
1. 安装系统依赖
2. 自动检测 Python 版本，Ubuntu 20.04 不满足时自动安装 Python 3.11
3. 创建虚拟环境并安装项目依赖（使用清华镜像源加速）
4. 写入访问密码
5. 配置 systemd 服务（开机自启、崩溃自动重启）
6. 配置 Nginx 反向代理

> **服务器上有宝塔时 Nginx 会冲突**，改用以下命令跳过 Nginx：
> ```bash
> bash deploy/setup_server.sh --no-nginx
> ```
> 然后在宝塔面板 → 网站 → 反向代理，目标填 `http://127.0.0.1:7070`。

部署完成后在宝塔防火墙放行 **7070** 端口，浏览器访问：
```
http://你的服务器IP:7070
```

> **Ubuntu 版本说明**
> - Ubuntu 22.04 / 24.04：自带 Python 3.10+，直接可用
> - Ubuntu 20.04：自带 Python 3.8，脚本会自动安装 Python 3.11

---

## 四、访问密码

部署完成后，首次打开页面会看到登录界面。

**默认密码：`0391`**

登录后即可正常使用。如需修改密码，编辑部署脚本顶部后重新运行：

```bash
ACCESS_PASSWORD="新密码"    # 留空则关闭密码保护
```

> 本地运行（`deploy.bat`）默认不启用密码，无需登录。

---

## 五、加载模型

登录后点击**「加载模型」**，选择以下任一模型：

| 模型 | 参数量 | 说明 |
|---|---|---|
| Kronos-mini | 4.1M | 最快，适合快速预测 |
| Kronos-small | 24.7M | 均衡，推荐日常使用 |
| Kronos-base | 102.3M | 最准，需要较多内存 |

首次加载会从 HuggingFace 镜像自动下载，**需要等待几分钟**，下载完后缓存到本地，之后秒加载。

---

## 六、配置数据源

点击页面右上角**「配置数据源」**按钮：

1. 填写 **Tushare Token**（用于股票数据）
2. 填写 **天勤（TQSdk）账号和密码**（用于期货数据）
3. 点击「品种库 → 立即同步」，拉取全市场期货品种

配置保存在 `webui/datasource_config.json`，重启服务后自动读取。

## 七、日常运维

```bash
# 查看运行状态
systemctl status kronos

# 重启服务
systemctl restart kronos

# 停止服务
systemctl stop kronos

# 实时查看日志
tail -f /var/log/kronos/app.log

# 查看错误日志
tail -f /var/log/kronos/error.log
```

## 八、常见问题

**Q：脚本报 `invalid option` 或 `\r` 错误？**

Windows 换行符问题，执行以下命令转换后重新运行：
```bash
sed -i 's/\r//' deploy/setup_server.sh && bash deploy/setup_server.sh
```

**Q：有宝塔时 Nginx 启动失败（80 端口冲突）？**

```bash
systemctl stop nginx && systemctl disable nginx
bash deploy/setup_server.sh --no-nginx
```
然后在宝塔面板配置反向代理，目标 URL：`http://127.0.0.1:7070`。

**Q：模型列表为空或加载失败？**

缺少依赖包，手动安装：
```bash
/root/Kronos_Deploy/.venv/bin/pip install einops safetensors tqdm \
    -i https://pypi.tuna.tsinghua.edu.cn/simple
systemctl restart kronos
```

**Q：服务启动失败，如何排查？**

```bash
journalctl -u kronos -n 50
tail -50 /var/log/kronos/error.log
```

**Q：服务器重启后服务会自动启动吗？**

会。`setup_server.sh` 已执行 `systemctl enable kronos`，开机自动拉起。

**Q：忘记访问密码怎么办？**

直接编辑配置文件：
```bash
nano ~/Kronos_Deploy/webui/datasource_config.json
```
找到 `access_password` 字段修改为新密码，保存后重启服务：
```bash
systemctl restart kronos
```
