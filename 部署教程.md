# Kronos 部署与调试完整教程

---

## 一、本地运行（Windows）

### 1.1 一键启动

双击项目根目录下的 `deploy.bat`，脚本自动完成：

1. 检测并创建 Python 虚拟环境（`.venv`）
2. 安装所有依赖
3. 启动 Web 服务并自动打开浏览器

访问地址：`http://localhost:7070`

> 本地运行**不需要输入密码**，直接进入主界面。

### 1.2 本地出现登录页怎么办

原因：之前测试时曾通过 UI 设置了访问密码，进程内存里仍保留旧值。

**解决方法一：重启 app（推荐）**
关掉 deploy.bat 窗口，重新双击运行即可。

**解决方法二：从 UI 关闭密码**
登录后进入「配置数据源 → 访问密码」标签，输入框留空，点「保存」，重启后生效。

**解决方法三：直接清空配置文件里的密码**
```bash
# 编辑配置文件，删除 access_password 字段或将其值改为空字符串
notepad webui\datasource_config.json
```

## 二、云服务器部署（Ubuntu）

### 2.1 环境要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Ubuntu 20.04 / 22.04 / 24.04 |
| Python | 3.9 及以上（脚本自动安装） |
| 内存 | 建议 2GB 以上（加载模型需 4GB+）|
| 端口 | 7070（Flask）、80（Nginx，可选）|

### 2.2 连接服务器

```bash
ssh root@你的服务器IP
```

### 2.3 上传项目

**本地 PowerShell 执行：**

```powershell
# 进入项目上级目录
cd C:\Users\...\你的路径

# 压缩
Compress-Archive -Path Kronos_Deploy -DestinationPath kronos.zip -Force

# 上传
scp kronos.zip root@你的服务器IP:~
```

**服务器上解压：**

```bash
apt install unzip -y
unzip kronos.zip
```

### 2.4 一键部署

```bash
cd ~/Kronos_Deploy
sed -i 's/\r//' deploy/setup_server.sh
bash deploy/setup_server.sh
```

> `sed` 命令用于去除 Windows 换行符（`\r`），必须执行，否则脚本报错。

脚本自动完成：
1. 安装系统依赖（curl、unzip 等）
2. 检测 Python 版本，Ubuntu 20.04 时自动安装 Python 3.11
3. 创建虚拟环境，安装项目依赖（使用清华镜像源）
4. 写入默认访问密码 `0391`
5. 配置 systemd 服务（开机自启、崩溃自动重启）
6. 配置 Nginx 反向代理（80 端口）

**有宝塔面板时（Nginx 冲突）：**

```bash
bash deploy/setup_server.sh --no-nginx
```

然后在宝塔面板 → 网站 → 反向代理，目标地址填 `http://127.0.0.1:7070`。

部署完成后放行防火墙 **7070** 端口，浏览器访问：

```
http://你的服务器IP:7070
```

**Ubuntu 版本说明：**
- Ubuntu 22.04 / 24.04：自带 Python 3.10+，直接可用
- Ubuntu 20.04：自带 Python 3.8，脚本自动安装 Python 3.11

## 三、访问密码

### 3.1 默认密码

服务器部署后默认密码为 `0391`，登录后即可使用。

> 本地运行（`deploy.bat`）默认**不启用**密码。

### 3.2 修改密码（推荐方式：UI）

登录后点击「配置数据源 → 访问密码」标签：
- 输入新密码 → 保存 → 重启服务生效
- **留空保存** → 关闭密码保护

### 3.3 修改密码（命令行）

```bash
nano ~/Kronos_Deploy/webui/datasource_config.json
```

找到 `access_password` 字段，修改值，保存后重启：

```bash
systemctl restart kronos
```

### 3.4 忘记密码

```bash
# 直接清空密码（删除该字段或置为空字符串）
python3 -c "
import json
path = '/root/Kronos_Deploy/webui/datasource_config.json'
cfg = json.load(open(path))
cfg.pop('access_password', None)
json.dump(cfg, open(path,'w'), ensure_ascii=False)
print('密码已清除')
"
systemctl restart kronos
```

## 四、加载模型

登录后点击**「加载模型」**，选择模型：

| 模型 | 参数量 | 说明 |
|---|---|---|
| Kronos-mini | 4.1M | 最快，适合快速预测 |
| Kronos-small | 24.7M | 均衡，推荐日常使用 |
| Kronos-base | 102.3M | 最准，需要较多内存 |

首次加载从 HuggingFace 镜像自动下载，需等待几分钟，之后缓存到本地秒加载。

## 五、配置数据源

点击页面右上角**「配置数据源」**：

1. **A股数据**：填写 Tushare Token
2. **期货数据**：填写天勤（TQSdk）账号和密码
3. **品种库**：点击「立即同步」拉取全市场期货品种

配置保存在 `webui/datasource_config.json`，重启后自动读取。

---

## 六、数据文件管理

页面左侧「选择数据文件」区域支持：

- **上传文件**：点击「⬆ 上传文件」，支持 `.csv` 和 `.feather` 格式，上传后自动出现在下拉列表
- **删除文件**：选中通过上传添加的文件后，「🗑 删除」按钮变为可用，点击确认后删除
- **刷新列表**：点击「↻ 刷新列表」重新扫描文件目录

> 内置数据文件（`data/` 目录）为只读，不可删除，下拉列表中显示 `[只读]` 标记。

## 八、日常运维

```bash
# 查看运行状态
systemctl status kronos

# 重启服务
systemctl restart kronos

# 停止服务
systemctl stop kronos
```

## 九、常见问题排查

**Q：脚本报 `invalid option` 或 `\r` 错误？**

Windows 换行符问题，执行以下命令后重新运行：
```bash
sed -i 's/\r//' deploy/setup_server.sh && bash deploy/setup_server.sh
```

**Q：有宝塔时 Nginx 启动失败（80 端口冲突）？**

```bash
systemctl stop nginx && systemctl disable nginx
bash deploy/setup_server.sh --no-nginx
```
然后在宝塔面板配置反向代理，目标 URL：`http://127.0.0.1:7070`。

**Q：模型列表为空或加载失败？**

缺少依赖包，手动安装：
```bash
/root/Kronos_Deploy/.venv/bin/pip install einops safetensors tqdm \
    -i https://pypi.tuna.tsinghua.edu.cn/simple
systemctl restart kronos
```

**Q：服务启动失败，如何排查？**

```bash
# 查看详细错误
journalctl -u kronos -n 50
tail -50 /var/log/kronos/error.log
```

常见原因：
- 端口 7070 被占用：`lsof -i:7070`，`kill -9 <PID>`
- Python 依赖缺失：重新运行 `bash deploy/setup_server.sh`
- 配置文件损坏：`cat ~/Kronos_Deploy/webui/datasource_config.json`

**Q：服务器重启后服务会自动启动吗？**

会。`setup_server.sh` 已执行 `systemctl enable kronos`，开机自动拉起。

**Q：期货品种同步失败？**

确认天勤账号已配置，且账号有效。同步需要联网访问天勤服务器，服务器需要能访问外网。

**Q：本地删除文件按钮是灰色的？**

只有通过「⬆ 上传文件」添加到 `uploads/` 目录的文件才可删除。项目自带的数据文件为只读，不支持从 UI 删除。
