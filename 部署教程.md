# Kronos 云服务器部署教程

## 环境要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Ubuntu 20.04 / 22.04 / 24.04 |
| Python | 3.9 及以上（脚本自动安装） |
| 内存 | 建议 2GB 以上（加载模型需 4GB+）|
| 端口 | 80（Nginx）、7070（Flask，内网） |

---

## 一、连接服务器

在本地打开终端（Windows 用 PowerShell 或 CMD）：

```bash
ssh root@你的服务器IP
```

---

## 二、上传项目到服务器

**方式 A：压缩包上传（推荐）**

本地 PowerShell 执行：
```powershell
# 压缩项目
Compress-Archive -Path Kronos_Deploy -DestinationPath kronos.zip

# 上传到服务器
scp kronos.zip root@你的服务器IP:/opt/
```

服务器上执行：
```bash
cd /opt
apt install unzip -y
unzip kronos.zip
mv Kronos_Deploy kronos
```

**方式 B：Git 克隆**

```bash
cd /opt
git clone 你的仓库地址 kronos
```

---

## 三、一键部署

根据服务器环境选择对应方式：

---

### 方式 A：Ubuntu 纯命令行（推荐）

```bash
cd /opt/kronos
bash deploy/setup_server.sh
```

脚本自动完成：
1. 安装系统依赖
2. 自动检测 Python 版本，Ubuntu 20.04 不满足时自动安装 Python 3.11
3. 创建虚拟环境并安装项目依赖
4. 写入访问密码
5. 配置 systemd 服务（开机自启、崩溃自动重启）
6. 配置 Nginx 反向代理
7. 启动服务

> **不需要 Nginx 时**（直接用 IP:7070 访问）：
> ```bash
> bash deploy/setup_server.sh --no-nginx
> ```

部署完成后浏览器访问 `http://你的服务器IP` 即可。

---

### 方式 B：宝塔面板

**第一步：上传项目**

在宝塔终端执行：
```bash
cd /www/wwwroot
# 解压之前上传的压缩包
unzip /tmp/kronos.zip && mv Kronos_Deploy kronos
# 或 Git 克隆
git clone 你的仓库地址 kronos
```

**第二步：运行宝塔专用部署脚本**

```bash
bash /www/wwwroot/kronos/deploy/setup_baota.sh
```

脚本自动完成：
1. 查找宝塔内置 Python 并创建虚拟环境
2. 安装项目依赖
3. 在系统 Python 上安装 tqsdk（期货数据子进程使用）
4. 写入访问密码
5. 配置 Supervisor 守护进程（开机自启）
6. 配置 Nginx 反向代理

**第三步：完成**

浏览器访问 `http://你的服务器IP` 即可。

宝塔常用运维命令：
```bash
supervisorctl status kronos       # 查看状态
supervisorctl restart kronos      # 重启
tail -f /var/log/kronos/app.log   # 查看日志
```

---

> **Ubuntu 版本说明**
> - Ubuntu 22.04 / 24.04：自带 Python 3.10+，直接可用
> - Ubuntu 20.04：自带 Python 3.8，脚本会自动从 deadsnakes PPA 安装 Python 3.11

## 三、访问密码

部署完成后，首次打开页面会看到登录界面。

**默认密码：`0391`**

登录后即可正常使用。如需修改密码，在部署脚本顶部修改后重新运行：

```bash
# setup_server.sh 或 setup_baota.sh 顶部配置项
ACCESS_PASSWORD="新密码"    # 留空则关闭密码保护
```

> 本地运行（`deploy.bat`）默认不启用密码，无需登录。

---

## 四、配置数据源

首次访问后，点击页面右上角**「配置数据源」**按钮：

1. 填写 **Tushare Token**（用于股票数据）
2. 填写 **天勤（TQSdk）账号和密码**（用于期货数据）
3. 点击「品种库 → 立即同步」，拉取全市场期货品种

配置保存在 `webui/datasource_config.json`，重启服务后自动读取。

## 五、日常运维

使用 `manage.sh` 脚本管理服务：

```bash
# 查看运行状态
bash deploy/manage.sh status

# 实时查看日志
bash deploy/manage.sh log

# 查看错误日志
bash deploy/manage.sh errlog

# 重启服务
bash deploy/manage.sh restart

# 停止服务
bash deploy/manage.sh stop

# 更新代码后重启（git 部署时使用）
bash deploy/manage.sh update
```

也可直接使用 systemctl：

```bash
systemctl status kronos
systemctl restart kronos
systemctl stop kronos
```

日志文件位置：

```
/var/log/kronos/app.log     # 应用日志
/var/log/kronos/error.log   # 错误日志
```

## 六、修改服务端口

默认端口为 **7070**，如需修改：

编辑 systemd 服务文件：

```bash
nano /etc/systemd/system/kronos.service
```

找到以下行，修改端口号：

```ini
Environment=KRONOS_PORT=7070
```

同步修改 Nginx 配置（`/etc/nginx/sites-available/kronos`）中的 `proxy_pass` 端口，然后重载：

```bash
systemctl daemon-reload
systemctl restart kronos
systemctl reload nginx
```

## 七、配置 HTTPS（可选）

有域名时可申请免费 SSL 证书：

```bash
# 安装 certbot
apt install certbot python3-certbot-nginx

# 申请证书（替换为你的域名）
certbot --nginx -d your-domain.com
```

申请成功后 Nginx 会自动配置 HTTPS，证书每 90 天自动续期。

参考配置见 `deploy/nginx.conf` 文件中的注释部分。

## 八、常见问题

**Q：服务启动失败，如何排查？**

```bash
# 查看详细报错
journalctl -u kronos -n 50
bash deploy/manage.sh errlog
```

**Q：TQSdk 无法使用怎么办？**

服务器上需要单独安装 TQSdk：

```bash
/opt/kronos/.venv/bin/pip install tqsdk
```

安装后在页面「配置数据源」中填写天勤账号，点击「立即同步」验证。

**Q：如何查看当前占用 7070 端口的进程？**

```bash
ss -tlnp | grep 7070
```

**Q：服务器重启后服务会自动启动吗？**

会。`setup_server.sh` 已执行 `systemctl enable kronos`，开机自动拉起。

**Q：忘记访问密码怎么办？**

在服务器上直接编辑配置文件：

```bash
# 纯命令行部署
nano /opt/kronos/webui/datasource_config.json

# 宝塔部署
nano /www/wwwroot/kronos/webui/datasource_config.json
```

找到 `access_password` 字段修改为新密码，保存后重启服务生效。
