## 1. 项目介绍

### 1.1 什么是 Kronos

Kronos 是首个开源的**金融 K 线基础模型**，在全球 45 个交易所的历史数据上预训练。采用两阶段架构：

1. **专用分词器（KronosTokenizer）**：将连续的 OHLCV 数据量化为离散 token，类似 LLM 的 BPE 分词
2. **自回归 Transformer（Kronos）**：像 GPT 一样在 token 序列上做下一步预测，完成多步 K 线预测

**核心优势**：
- 预训练数据覆盖面广，无需训练即有基础预测能力
- 支持精调，适应特定市场/品种
- 同时预测 OHLCV 所有字段，不只预测收盘价

### 1.2 模型家族

| 模型 | 分词器 | 最大上下文 | 参数量 | 适用场景 |
|------|--------|-----------|--------|---------|
| Kronos-mini | Kronos-Tokenizer-2k | **2048** | 4.1M | 长序列快速预测 |
| Kronos-small | Kronos-Tokenizer-base | 512 | 24.7M | 日常使用推荐 |
| Kronos-base | Kronos-Tokenizer-base | 512 | 102.3M | 高精度场景 |
| Kronos-large | Kronos-Tokenizer-base | 512 | 499.2M | 暂未开源 |

> **推荐**：日常使用 Kronos-small（平衡速度与精度）；有 GPU 且追求精度用 Kronos-base；序列很长时用 Kronos-mini（支持 2048 上下文）。

### 1.3 本版本新增功能（二次改版）

- **A 股日线数据**：通过 Tushare API 直接输入 6 位代码拉取（需 Token）
- **期货全周期数据**：通过 TQSdk（天勤）拉取，支持日K、1小时K、30分钟K、15分钟K
- **正确的分钟K线边界**：下载1分钟原始数据后自动重采样，K线边界与交易软件完全一致（正确处理 10:15~10:30、午休等交易时段间隔）
- **中文名搜索**：代码输入框支持直接输入中文品种名（如"螺纹钢"）自动搜索匹配
- **自动同步品种库**：通过天勤 TQSdk 一键拉取全市场期货品种，代码识别和中文名搜索自动更新
- **简短期货代码**：输入 `rb2610`、`SR607`、`PK610` 等，自动补全交易所前缀
- **图表品种名称**：图表标题、图例自动显示中文品种名（如 `rb2610 螺纹钢`）
- **无模型降级**：未加载 Kronos 时自动降级为技术指标简化预测，仍可看到图表

## 2. 环境安装

### 2.1 系统要求

- Python 3.9 ~ 3.11（推荐 3.11）
- CUDA 11.8+（有 GPU 时）或 CPU 运行（较慢）
- 内存 8GB+，显存 4GB+（推荐）

### 2.2 安装基础依赖

```bash
# 克隆项目（或直接使用已有目录）
cd Kronos_Deploy

# 安装模型核心依赖
pip install -r requirements.txt

# 安装 Web 界面依赖
pip install -r webui/requirements.txt
```

**requirements.txt 核心包**：
```
numpy
pandas==2.2.2
torch>=2.1.0
einops==0.8.1
huggingface_hub==0.33.1
matplotlib==3.9.3
safetensors==0.6.2
tqdm==4.67.1
```

**webui/requirements.txt**：
```
flask==2.3.3
flask-cors==4.0.0
plotly==5.17.0
tushare        # A 股数据
tqsdk          # 期货数据
```

### 2.3 国内网络配置（重要）

模型托管在 HuggingFace，国内需配置镜像：

```bash
# 方法一：环境变量（临时）
export HF_ENDPOINT=https://hf-mirror.com

# 方法二：写入系统（永久，Windows PowerShell）
[System.Environment]::SetEnvironmentVariable("HF_ENDPOINT", "https://hf-mirror.com", "User")
```

项目代码已在 `app.py` 顶部自动配置：
```python
os.environ['HF_ENDPOINT'] = 'https://hf-mirror.com'
```

### 2.4 验证安装

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -c "import tushare; print(tushare.__version__)"
python -c "import tqsdk; print('tqsdk OK')"
```

## 3. Web 界面使用

### 3.1 启动服务

```bash
cd webui
python app.py
# 或
python run.py
```

服务启动后访问：**http://localhost:7070**

启动日志中会显示 TQSdk 使用的 Python 解释器：
```
[TQSdk] using Python: C:\Users\...\Python311\python.exe
```

> **注意**：如果 `[TQSdk] using Python:` 后面显示的是 venv 路径，说明系统 Python 没有正确找到 tqsdk，需要在系统 Python 中单独安装 `pip install tqsdk`。

---

### 3.2 配置数据源

点击控制面板中的 **配置数据源** 按钮，弹窗分三个 Tab：

#### A股数据 Tab

1. 前往 [tushare.pro](https://tushare.pro) 注册并获取 Token
2. 在"Tushare Token"输入框粘贴 Token，点击**保存**
3. 状态徽章变为绿色"已配置"即成功，主界面状态栏同步更新

> Tushare 免费版有接口调用频率限制，日线数据一般够用。积分 120 以上账号可访问 `pro_bar` 接口。

#### 期货数据 Tab

1. 前往 [shinnytech.com](https://www.shinnytech.com) 注册天勤账号（需实名）
2. 填入**天勤用户名**和**密码**，点击**保存**
3. 状态徽章变为绿色"已配置"即成功

> 天勤 TQSdk 每月有免费数据额度，日线数据不限量，分钟线有一定用量限制。

#### 品种库 Tab

- **立即同步**：从天勤拉取全市场所有期货品种，自动更新代码识别和中文名搜索库（约 10~30 秒）。同步后显示品种数量和最后同步时间。
- **手动新增品种**：遇到不认识的新合约代码时，填写品种代码、交易所、中文名，点击**保存品种**手动添加。

### 3.3 加载模型

左侧面板"模型选择"区域：

1. **选择模型**：下拉选择 kronos-mini / kronos-small / kronos-base
2. **选择设备**：有 GPU 选 CUDA:0，否则选 CPU
3. 点击**加载模型**，等待下载（首次需要下载，约几百 MB）

加载成功后按钮变绿，显示"模型就绪"。

> **未加载模型时**：系统会降级为"简化技术分析"模式（基于 MA/RSI 的简单预测），仍然可以生成图表，但预测质量有限。

### 3.4 股票预测（按代码）

在"代码（股票/期货）"输入框中输入 6 位数字，如 `600036`：

- 首位 6 或 9 → 上交所（自动加 .SH）
- 其余 → 深交所（自动加 .SZ）

**参数说明**：

| 参数 | 默认值 | 建议值 | 说明 |
|------|--------|--------|------|
| 回溯交易日数 | 100 | 100～250 | 拉取最近 N 个交易日历史数据，100≈5个月，250≈1年 |
| 预测交易日数 | 22 | 22 | 约 1 个月，范围 5～60 |
| 预测温度 T | 1.0 | 1.0 ~ 1.2 | 越大越随机，0.8 偏保守 |
| 核采样 top_p | 0.9 | 0.9 ~ 1.0 | 控制预测多样性 |
| 样本数量 | 1 | 2 ~ 3 | 多次取均值，质量更稳 |

点击**按代码直接预测**，等待数秒出现 K 线预测图。

### 3.6 分钟/小时 K 线预测

在"K线周期"下拉框中选择所需周期，**仅期货支持分钟/小时线**：

| 周期选项 | 说明 |
|---------|------|
| 日K | 股票和期货均支持 |
| 1小时K | 仅期货，需天勤 TQSdk 账号 |
| 30分钟K | 仅期货，需天勤 TQSdk 账号 |
| 15分钟K | 仅期货，需天勤 TQSdk 账号 |

**K 线边界说明**：

系统通过下载1分钟原始数据并自动重采样来生成各周期 K 线，可确保 K 线时间边界与主流交易软件（文华财经、同花顺等）完全一致。以下交易时段间隔均被正确处理：

| 间隔类型 | 示例 | 处理方式 |
|---------|------|---------|
| 日间短暂休市（<4小时） | 10:15~10:30（15分钟）| 跨越休市连续计数 |
| 午休（<4小时） | 11:30~13:30（2小时）| 跨越午休连续计数 |
| 隔夜间隔（≥4小时） | 夜盘结束到次日开盘 | 重新开始新的交易日计数 |

> 以 JD（鸡蛋，DCE）30分钟线为例：正确输出 09:01、09:31、10:01、10:31（跨越10:15~10:30休市）、11:01、11:31、13:31、14:01、14:31、15:01，与交易软件完全一致。

**分钟/小时线参数说明**：

| 参数 | 默认值 | 建议范围 | 说明 |
|------|--------|---------|------|
| 回溯K线条数 | 400 | 固定400 | 与模型上下文窗口匹配，不建议修改 |
| 预测K线条数 | 24 | 10～48 | 见下表参考 |

**预测条数参考**：

| 周期 | 默认24条 = | 建议上限 |
|------|-----------|---------|
| 1小时K | 约3个交易日 | 48条（约6天） |
| 30分钟K | 约1.5个交易日 | 48条（约3天） |
| 15分钟K | 约6小时 | 48条（约1.5天） |

> 预测跨度越长误差越大。分钟线建议预测不超过2～3天，超出范围的预测仅供参考。

### 3.7 本地 CSV 文件预测

适合自有数据（不依赖 Tushare/TQSdk）：

**步骤**：
1. 准备符合格式的 CSV 文件（见第 6 节）
2. 将文件放入 `data/` 目录（或项目内 `examples/data/`）
3. 界面右侧"数据文件"区域点击刷新，选择文件
4. 点击"加载数据"，右侧会显示数据概览（行数、时间范围、价格范围等）
5. 拖动时间窗口滑块，选择预测起始位置（绿色区域 = 400 历史 + 120 预测共 520 个数据点）
6. 点击"开始预测"（需提前加载模型）

### 3.8 控制面板参数详解

#### 一、模型区（左上角）

| 控件 | 说明 |
|------|------|

#### 二、按代码预测区（中间）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|

#### 三、文件上传区（右侧）

| 控件 | 说明 |
|------|------|
| **选择数据文件** | 读取 `webui/data/` 目录下的 CSV/Excel 文件 |

| **预测长度** | 固定 120（文件模式不可改） |

---

#### 四、高级参数区（底部横排）

底部一行横向排列，两个滑块各自显示参数名、当前值和说明：

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| **样本数量** | 1 | 1～5 | 生成多条预测曲线，多条取均值可提升稳定性 |
| **预测温度 T** | 1.0 | 0.1～2.0 | 控制随机性：低→保守平滑，高→波动多样 |
| **核采样 top_p** | 0.9 | 0.1～1.0 | 限制采样范围：低→只选高概率走势，高→更开放 |

**Temperature 使用建议**：

| 数值 | 效果 |
|------|------|
| 0.5 ~ 0.8 | 保守，曲线平滑，趋势延续性强 |
| 1.0 | 默认，平衡（推荐） |
| 1.2 ~ 2.0 | 激进，波动更大，适合观察极端情形 |

**top_p 使用建议**：

| 数值 | 效果 |
|------|------|
| 0.6 | 严格，只选概率最高的走势 |
| 0.9 | 默认，较宽泛（推荐） |
| 1.0 | 完全开放，最多样化 |

### 3.9 期货品种库管理

#### 自动同步（推荐）

打开"配置数据源"弹窗 → 切换到**品种库** Tab → 点击**立即同步**。

系统调用天勤 TQSdk 拉取全市场所有期货品种（需已配置天勤账号），约 10~30 秒完成。同步后：
- 所有期货品种均可通过代码或中文名搜索
- 即使交易所新增或调整交易时段，也能正确识别

同步记录（品种数量和最后同步时间）会显示在按钮下方。

#### 手动新增

当遇到系统尚未识别的期货品种代码时（如新上市合约），可手动补充：

| 字段 | 说明 | 示例 |
|------|------|------|
| **品种代码（不含月份）** | 品种的根代码，小写字母 | `pp`、`bz`、`br` |
| **所属交易所** | 下拉选择对应交易所 | DCE（大商所）|
| **中文名称** | 品种名称，用于图表显示（可选） | 聚丙烯 |

填写后点击**保存品种**，之后即可直接输入该品种的合约代码（如 `pp2609`）进行预测。

> 手动新增的品种信息保存在 `webui/futures_symbols.json`，重启服务后仍然有效。

#### 已内置交易所对照

| 交易所 | 代表品种 |
|--------|---------|
| SHFE（上期所） | rb螺纹钢、cu铜、au黄金、ru天然橡胶、ao氧化铝、ec集运欧线 |
| INE（能源中心） | sc原油 |

## 4. Python 编程接口

适合批量研究或集成到自己的策略系统。

### 4.1 基础预测

```python
import sys
sys.path.append("path/to/Kronos_Deploy")

import pandas as pd
from model import Kronos, KronosTokenizer, KronosPredictor

# 1. 加载分词器和模型（首次自动从 HuggingFace 下载）
tokenizer = KronosTokenizer.from_pretrained("NeoQuasar/Kronos-Tokenizer-base")
model = Kronos.from_pretrained("NeoQuasar/Kronos-small")

# 2. 创建预测器
predictor = KronosPredictor(
    model,
    tokenizer,
    device="cuda:0",   # 无 GPU 改为 "cpu"
    max_context=512,   # Kronos-mini 改为 2048
    clip=5             # 标准化裁剪范围（一般不用改）
)

# 3. 准备历史数据
df = pd.read_csv("./examples/data/XSHG_5min_600977.csv")
df['timestamps'] = pd.to_datetime(df['timestamps'])

lookback = 400    # 输入历史 K 线数量
pred_len = 120    # 预测未来 K 线数量

x_df         = df.loc[:lookback-1, ['open', 'high', 'low', 'close', 'volume', 'amount']]
x_timestamp  = df.loc[:lookback-1, 'timestamps']
y_timestamp  = df.loc[lookback:lookback+pred_len-1, 'timestamps']

# 4. 预测
pred_df = predictor.predict(
    df=x_df,
    x_timestamp=x_timestamp,
    y_timestamp=y_timestamp,
    pred_len=pred_len,
    T=1.0,            # 温度
    top_p=0.9,        # Top-P 核采样
    top_k=0,          # Top-K（0 = 不限制）
    sample_count=1,   # 采样次数，>1 时取均值
    verbose=True
)

# 5. 使用结果
print(pred_df.head())
# 输出示例：
#              open    high     low   close   volume     amount
# 2024-01-03  10.65   10.80   10.55   10.72  1100000  11792000
# 2024-01-04  10.72   10.90   10.68   10.85  1050000  11392500
# ...
```

### 4.3 批量预测

同时预测多个股票/合约，效率更高：

```python
# 准备 5 个样本的数据
dfs, x_tsp_list, y_tsp_list = [], [], []

for i in range(5):
    start = i * 400
    sub_df   = df.loc[start:start+lookback-1, ['open','high','low','close','volume','amount']]
    x_tsp    = df.loc[start:start+lookback-1, 'timestamps']
    y_tsp    = df.loc[start+lookback:start+lookback+pred_len-1, 'timestamps']
    dfs.append(sub_df)
    x_tsp_list.append(x_tsp)
    y_tsp_list.append(y_tsp)

pred_list = predictor.predict_batch(
    df_list=dfs,
    x_timestamp_list=x_tsp_list,
    y_timestamp_list=y_tsp_list,
    pred_len=pred_len,
    T=1.0,
    top_p=0.9,
    sample_count=1,
    verbose=True
)

for i, pred in enumerate(pred_list):
    print(f"样本 {i} 预测结果：{pred['close'].values}")
```

**批量预测注意事项**：
- 所有样本的 `lookback` 和 `pred_len` 必须相同
- 不同股票可以放在同一批次
- 批量处理比逐个预测快 3～5 倍

### 4.4 参数调优建议

| 场景 | T | top_p | sample_count |
|------|---|-------|-------------|
| 保守预测（趋势跟随） | 0.8 | 0.85 | 1 |
| 标准使用 | 1.0 | 0.9 | 1 |
| 探索多种可能性 | 1.2 | 0.95 | 3 |
| 高质量平均预测 | 1.0 | 0.9 | 5 |

## 5. 模型精调

精调可以让 Kronos 更好地适应特定市场（A 股、期货）或特定品种，通常能显著提升预测精度。

### 5.1 精调流程概览

```
原始数据（Qlib 格式）
       ↓
① qlib_data_preprocess.py   —— 数据预处理，生成 pkl 训练集
       ↓
② train_tokenizer.py        —— 精调分词器（学习本市场量化方式）
       ↓
③ train_predictor.py        —— 精调预测器（基于精调分词器学习预测）
       ↓
④ qlib_test.py              —— 回测评估预测效果
```

### 5.2 准备精调环境

```bash
# 额外安装精调依赖
pip install pyqlib
pip install comet_ml       # 可选，用于实验记录

# 下载 Qlib A 股数据（约 1.5GB）
python -c "import qlib; from qlib.utils import init_instance_by_config; qlib.init(provider_uri='~/.qlib/qlib_data/cn_data', region='cn')"
# 或手动下载：https://github.com/microsoft/qlib#data-preparation
```

### 5.3 配置参数

编辑 `finetune/config.py`，核心参数说明：

```python
# ─── 数据相关 ───
qlib_data_path = "~/.qlib/qlib_data/cn_data"   # Qlib 数据目录
instrument = 'csi300'     # 股票池，可选 csi500、all 等

# ─── 时间范围 ───

dataset_end_time   = "2025-06-05"
train_time_range   = ["2011-01-01", "2022-12-31"]   # 训练集

```

### 5.4 数据预处理

```bash
cd finetune
python qlib_data_preprocess.py
```

脚本会：
1. 从 Qlib 加载指定股票池的历史数据
2. 按时间拆分为训练/验证/测试集
3. 对每只股票做日线数据清洗（去 NaN、排序）
4. 保存为 `train_data.pkl`、`val_data.pkl`、`test_data.pkl`

**数据处理逻辑**：
```
原始 OHLCV → 去除全零行 → 时间升序排列 → 按股票保存字典
```

### 5.5 精调分词器

分词器学习如何把 OHLCV 数据量化为离散 token，精调后能更好表达 A 股/期货的价格分布特征。

```bash
cd finetune

# 单 GPU
python train_tokenizer.py

# 多 GPU（替换 2 为实际 GPU 数量）
torchrun --standalone --nproc_per_node=2 train_tokenizer.py
```

**训练过程**：
- 加载预训练 `KronosTokenizer`
- 冻结编码器权重，只微调 `BSQuantizer`（量化层）
- 损失 = 重建损失 + 量化损失

```
Loss = (mse(z_pre, x) + mse(z, x)) / 2 + bsq_loss
```

- 每轮在验证集上评估，自动保存最优检查点到 `save_path/tokenizer_best.pth`

**常见调整**：
- 显存不足：减小 `batch_size`（如 16）
- 精度不够：增加 `epochs`（如 50）
- 过拟合：增大 `adam_weight_decay`（如 0.2）

### 5.6 精调预测器

使用精调好的分词器，在 A 股数据上微调 Kronos 预测器。

```bash
cd finetune

# 单 GPU
python train_predictor.py

# 多 GPU
torchrun --standalone --nproc_per_node=2 train_predictor.py
```

**训练过程**：
1. 加载精调后的分词器（**冻结**，不再更新）
2. 加载预训练 Kronos 预测器
3. 在线将输入转为 token，计算交叉熵损失：

```
token_s1, token_s2 = tokenizer.encode(x, half=True)
logits_s1, logits_s2 = model(token_s1[:, :-1], token_s2[:, :-1], timestamps)
loss_s1 = cross_entropy(logits_s1, token_s1[:, 1:])
loss_s2 = cross_entropy(logits_s2, token_s2[:, 1:])
loss = (loss_s1 + loss_s2) / 2
```

4. 使用 `OneCycleLR` 学习率调度
5. 梯度裁剪（max_norm=3.0）防止梯度爆炸
6. 最优验证集模型保存到 `save_path/predictor_best.pth`

**精调时间参考**（单张 A100）：
- Kronos-small：约 2～4 小时（30 epochs，CSI300）
- Kronos-base：约 8～12 小时

### 5.7 回测评估

```bash
cd finetune
python qlib_test.py --device cuda:0
```

脚本会：
1. 在测试集上用精调模型做逐步推断
2. 根据预测价格变化生成多空信号
3. 执行 Top-K 策略回测（做多预测涨幅最大的 K 只）
4. 对比沪深300基准
5. 输出累计收益曲线图和性能指标（夏普率、最大回撤等）

结果保存在 `backtest_result_path/` 目录。

## 6. 数据格式规范

### 6.1 CSV 文件格式

**最简格式（仅 OHLC）**：
```csv
timestamps,open,high,low,close
2024-01-02,10.50,10.75,10.45,10.65
2024-01-03,10.65,10.80,10.60,10.75
```

**完整格式（OHLCV + 金额）**：
```csv
timestamps,open,high,low,close,volume,amount
2024-01-02,10.50,10.75,10.45,10.65,1000000,10650000
2024-01-03,10.65,10.80,10.60,10.75,1100000,11825000
```

**字段说明**：

| 字段 | 是否必须 | 类型 | 说明 |
|------|---------|------|------|
| timestamps | 可选 | datetime | 时间戳，缺失时自动生成 |
| open | **必须** | float | 开盘价 |
| high | **必须** | float | 最高价 |
| low | **必须** | float | 最低价 |
| close | **必须** | float | 收盘价 |
| volume | 可选 | float | 成交量，缺失时填 0 |
| amount | 可选 | float | 成交额，缺失时填 0 |

**字段命名兼容**：时间戳列支持 `timestamps`、`timestamp`、`date` 三种命名。

### 6.2 数据质量要求

- **不能有 NaN**（会导致预测报错）
- **时间必须升序**（旧→新）
- **最少行数**：建议至少 50 行，推荐 200 行以上
- **价格必须合理**：不能出现 `high < low` 等异常

### 6.3 模型内部数据处理流程

```
原始 OHLCV 数据
       ↓
按特征独立标准化：(x - mean) / (std + 1e-5)
       ↓
裁剪异常值到 [-5, 5]（clip=5）
       ↓
分词器编码为 token 序列
       ↓
Transformer 自回归预测
       ↓
反标准化：pred * std + mean
       ↓
输出预测 OHLCV DataFrame
```

## 7. 模型参数说明

### 7.1 预测参数详解

**Temperature（温度）**：
- 控制 token 采样的随机性
- 低温（0.5～0.8）：预测更集中、保守，趋势跟随
- 标准（1.0）：按模型学到的分布采样
- 高温（1.2～1.5）：预测更分散、探索性强

**Top-P（核采样）**：
- 只从累积概率 ≤ P 的 token 中采样
- 0.9：去掉最不可能的 10% 的情况
- 1.0：不过滤，保留全部可能性

**Sample Count（采样次数）**：
- 生成多个独立预测后取平均
- 相当于蒙特卡洛模拟，减少随机噪声
- 设为 3～5 次通常能明显提升预测稳定性，代价是时间加倍

### 7.2 分词器架构参数（BSQ 量化）

| 参数 | 含义 | 默认值 |
|------|------|--------|
| s1_bits | S1 量化位数，决定码本大小（2^8=256） | 8 |
| s2_bits | S2 量化位数 | 8 |
| beta | 提交损失权重（量化学习速度） | 0.99 |
| gamma0 | 每样本熵惩罚（防止码本坍缩） | 0.01 |
| gamma | 码本整体熵惩罚 | 0.01 |

### 7.3 Kronos 预测器架构参数

| 参数 | Kronos-mini | Kronos-small | Kronos-base |
|------|------------|-------------|------------|
| n_layers | 6 | 12 | 24 |
| d_model | 256 | 384 | 512 |
| n_heads | 4 | 6 | 8 |
| ff_dim | 1024 | 1536 | 2048 |
| max_context | 2048 | 512 | 512 |
| token_dropout_p | 0.1 | 0.1 | 0.1 |

## 8. 常见问题

### Q1：模型下载失败 / 连接 HuggingFace 超时

```bash
# 确认环境变量生效
echo $HF_ENDPOINT    # Linux/Mac
echo %HF_ENDPOINT%   # Windows CMD

# 手动设置镜像后重试
export HF_ENDPOINT=https://hf-mirror.com
python -c "from model import Kronos; Kronos.from_pretrained('NeoQuasar/Kronos-small')"
```

### Q2：期货数据拉取没反应（无报错无图表）

**排查步骤**：

1. 访问 `/api/sysinfo` 检查 TQSdk Python 路径：
   ```
   http://localhost:7070/api/sysinfo
   ```
   确认 `tqsdk_executable` 不是 venv 路径，且 `tqsdk_in_tqsdk_env` 有版本号。

2. 如果 `tqsdk_in_flask_env` 显示 NOT FOUND：
   ```bash
   # 在系统 Python（非 venv）中安装 tqsdk
   C:\Users\Administrator\AppData\Local\Programs\Python\Python311\python.exe -m pip install tqsdk
   ```

3. 重启 Flask 服务后重试。

4. 确认合约未到期：使用主力连续合约 `KQ.m@SHFE.rb` 最保险。

### Q3：CUDA 内存不足（Out of Memory）

- 换用 `Kronos-mini`（参数量最小）
- 减小 `sample_count` 为 1
- 减小 `lookback` 窗口长度
- 改用 CPU（慢但不受显存限制）

### Q4：预测结果看起来不合理（剧烈震荡或一直上涨）

- 降低 Temperature（试试 0.8～1.0）
- 增加 `sample_count` 到 3～5 取均值
- 检查输入数据质量（有无异常值、缺失值）
- 尝试更大的模型（Kronos-base vs Kronos-small）

### Q5：A 股数据下载失败

- 确认 Tushare Token 已正确配置（状态徽章显示绿色）
- 确认 Token 有效：用积分 120 以上的账号（pro_bar 需要积分）
- 检查网络：Tushare 数据服务器不受代理影响，但本地防火墙可能拦截

### Q6：分钟K线时间与交易软件不一致

- 确认天勤账号已正确配置（期货数据 Tab 状态徽章显示绿色）
- 系统通过1分钟数据重采样生成分钟K线，K线边界与文华财经、同花顺等主流软件一致
- 如仍有偏差，可先在品种库 Tab 点击**立即同步**更新品种交易时段信息

### Q7：精调时 GPU 利用率低

- 增大 `batch_size`（显存允许的情况下）
- 使用 `torchrun` 多 GPU 并行
- 检查数据加载是否成为瓶颈（`num_workers` 参数）

### Q8：回测夏普率很低

- 增加精调训练数据时间范围
- 增大 `sample_count` 减少预测噪声
- 调整 Top-K 策略的 K 值（仓位集中度）
- 考虑增加事务成本模拟（默认可能未计入手续费）
